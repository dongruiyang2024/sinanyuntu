# 司南云图官网初始化设计

日期：2026-06-10

## 目标

为“司南云图（杭州）科技有限公司”初始化一个可上线、可扩展的官方网站。官网首版采用 Next.js 架构，以单页首页完成完整品牌表达，同时在导航和代码结构上预留后续多页面扩展空间。

首版定位为“可信公司官网 + AI 云图品牌叙事”。页面要先建立公司可信度，再用“AI 信标、数据链、机会点、云图导航”的视觉和文案形成记忆点。

## 品牌依据

品牌资料来自 `/Users/dongruiyang/文件/司南云图/`：

- 横版组合优先用于官网页眉。
- 图形标和应用图标用于 favicon、移动端图标和局部视觉装饰。
- 品牌含义：A/AI 外轮廓代表外贸 AI 服务与工具软件；S 曲线代表 Sinan、外贸业务链路和数据流；贴线节点代表 AI 工具、数据触点和运营环节；金色顶点代表目标市场、机会点和司南所指方向。

品牌色：

- 星图蓝：`#4457E8`
- 港湾蓝灰：`#264E68`
- 云图青：`#18C5B1`
- 机会金：`#F2A51A`
- 深墨色：`#101928`

## 信息架构

官网首版采用“单页为主，导航预留多页结构”的方式。

顶部导航展示：

- 首页
- 能力图谱
- 服务场景
- 关于我们
- 联系我们

这些导航在首版中跳转到首页锚点。后续可以把每个锚点拆成独立页面，不需要重做首屏和品牌体系。

## 页面结构

### 1. 页眉

使用横版 logo。导航保持克制、清晰，右侧放主要行动按钮“预约沟通”或“联系我们”。

### 2. 首屏

首屏采用白底和浅色科技感背景。主标题方向：

> 让外贸团队看见机会，并沿着数据链行动

辅助文案表达“司南云图以 AI 工具和服务交付连接市场洞察、商品运营、客户触达与增长复盘”。

首屏右侧或背景区域使用云图/数据链视觉：线条、节点、机会金目标点。视觉应呼应 logo，不使用过重的深色科技风。

主要按钮：

- 预约沟通
- 查看能力图谱

### 3. 信任关键词

在首屏下方展示 3 到 4 个短句或关键词，用于快速说明公司能力边界：

- AI 工具软件
- 外贸业务链路
- 数据化运营
- 产品与服务落地

### 4. AI 能力图谱

展示司南云图围绕外贸增长提供的能力模块：

- 市场洞察：发现目标市场、机会品类和需求信号。
- 商品运营：辅助商品信息优化、内容生成和卖点表达。
- 客户触达：支持询盘、沟通、转化线索的效率提升。
- 数据复盘：用数据链路沉淀运营反馈，指导下一轮动作。

能力区的视觉语言继续使用节点、连线、路径和机会点。

### 5. 服务场景

展示适合客户理解的业务场景，而不是堆砌技术名词：

- 新市场探索
- 商品内容优化
- 外贸运营提效
- 增长数据复盘

每个场景用简短标题、说明和结果导向文案表达。

### 6. 产品 + 服务交付方式

说明司南云图不是只做咨询，也不是只卖工具，而是“产品能力 + 服务落地”的组合：

- 工具能力：AI、数据链、自动化辅助、内容生成。
- 服务落地：结合客户外贸业务流程配置方案。
- 持续优化：基于运营数据和目标市场反馈迭代。

### 7. 关于我们

用简洁公司介绍表达身份：

> 司南云图（杭州）科技有限公司专注外贸 AI 产品与服务，帮助外贸团队把分散的市场信号、商品信息、运营动作和增长数据连接成可执行的业务云图。

### 8. 联系入口

首版不接入后端表单。页面底部展示清晰联系区：

- 联系按钮
- 联系邮箱显示 `contact@sinanyuntu.com`，并集中放在配置数据里，便于正式信息确认后替换。
- 电话或企业微信位置显示“请通过邮箱预约沟通”，不展示未确认号码或二维码。
- 简短合作提示

后续可替换成真实表单、企业微信二维码或 CRM 线索提交。

## 视觉方向

采用“可信导航型”：

- 以白底和浅色背景为主，表达专业、稳定、可信。
- 品牌蓝和青色负责科技感与行动路径。
- 机会金只用于关键机会点、强调状态和小面积高亮。
- 深墨色用于标题、导航和主要文字。
- 避免过度深色、过度渐变、装饰性圆球和泛科技模板感。
- 页面模块之间保持足够留白，但信息密度要服务公司官网，不做空泛营销页。

## 技术架构

使用：

- Next.js App Router
- TypeScript
- Tailwind CSS

首版为静态官网：

- 不接 CMS。
- 不接表单后端。
- 不接数据库。
- 不引入复杂动画库。

建议文件组织：

- `app/`：页面、布局和全局样式。
- `components/`：页眉、页脚、首页各 section、品牌视觉组件。
- `public/brand/`：logo、mark、app icon 等品牌资产。
- `lib/` 或 `data/`：首页文案和模块数据，便于后续调整。

## 响应式要求

- 桌面端首屏为左右结构：左侧文案与行动按钮，右侧云图视觉。
- 移动端首屏上下堆叠，logo、标题、按钮和云图视觉不能重叠。
- 导航在移动端折叠或简化，保留主要联系按钮。
- 固定格式视觉组件使用稳定尺寸或响应式约束，避免内容变化导致布局跳动。

## 测试与验证

实现完成后需要验证：

- `npm run lint`
- `npm run build`
- 浏览器桌面视口检查首页首屏、导航、logo、能力区和联系区。
- 浏览器移动视口检查文本不溢出、不重叠，按钮可点击，品牌资产正常显示。

## 暂不包含

- 后端联系表单
- CMS 管理
- 登录、控制台或 SaaS 产品界面
- 多语言
- 博客
- 案例详情页
- 复杂滚动动画

这些内容后续可在当前官网结构上扩展。
